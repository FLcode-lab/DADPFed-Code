## Method Mapping to the Manuscript
The implementation corresponds to the core methodology described in the paper.

### DADPFed
Implemented components include:
- **Drift-aware dynamic mask**
  - $q^t = |h^t| / (|\theta^t| + \epsilon)$
  - Mask generated by quantile thresholding
- **Importance-aware retention**
  - Top-K retention based on parameter magnitude
  - Controlled by retention ratio $Th$
- **Combined masking**
  - $\tilde{\theta}^t = \theta^t \odot m^t \odot m_1^t$
- **Server-side drift tracking**: maintains global drift variable $h$
- **Client-side drift synchronization**: maintains local drift variable $g_i$

### DADPFed-SAM
In addition to the above, this variant applies a SAM-style two-step local optimization procedure on top of the DADPFed local objective.

## Implemented Options
The following method options are integrated into `train.py`:

- `--method DADPFed`
- `--method DADPFedSAM`

### DADPFed-related hyperparameters
- `--dadpfed-cycle`: Communication-cycle parameter $R$ (Default: 40)
- `--dadpfed-retention`: Importance-retention ratio $Th$ (Default: 0.75)
- `--dadpfed-mask-quantile`: Quantile threshold for drift-aware masking (Default: 0.75)
- `--momentum`: SGD momentum (Default: 0.9)

### Dataset support
- `--dataset UCF101-MA`

If needed, additional dataset configurations can be adapted from the existing framework.

## Step-by-Step Reproduction Pipeline
### Step 1. Install dependencies
```bash
pip install -r requirements.txt
```

### Step 2. Prepare datasets
Prepare the required benchmark data under the expected dataset directory structure used by the framework.

For martial-arts experiments:
- Reconstruct or prepare the relevant subset inputs for UCF101-MA
- Ensure that preprocessing can write cached split data to the expected path

In the current framework, the first run will decode/process videos and cache split data automatically.

Example cache location:
```
./Data/UCF101-MA_<clients>_<seed>_<split_rule>_<split_coef>/
```

### Step 3. Run a quick smoke test
The following commands provide a minimal verified run to check whether the code path is functioning correctly:
```bash
COMM_ROUNDS=1 LOCAL_EPOCHS=1 TOTAL_CLIENT=10 ACTIVE_RATIO=0.2 BATCH_SIZE=8 DATASET=UCF101-MA bash scripts/run_dadpfed.sh
COMM_ROUNDS=1 LOCAL_EPOCHS=1 TOTAL_CLIENT=10 ACTIVE_RATIO=0.2 BATCH_SIZE=8 DATASET=UCF101-MA bash scripts/run_dadpfedsam.sh
```

### Step 4. Run paper-style experiments
#### D1 setting: label-skew coefficient $\alpha_d = 0.3$
```bash
DATASET=UCF101-MA METHOD=DADPFed NON_IID=1 SPLIT_COEF=0.3 TOTAL_CLIENT=100 ACTIVE_RATIO=0.15 \
COMM_ROUNDS=1000 LOCAL_EPOCHS=5 DADPFED_CYCLE=40 DADPFED_RETENTION=0.75 \
bash scripts/run_dadpfed_exp.sh
```

#### D2 setting: $\alpha_d = 0.6$
```bash
DATASET=UCF101-MA METHOD=DADPFed NON_IID=1 SPLIT_COEF=0.6 TOTAL_CLIENT=100 ACTIVE_RATIO=0.15 \
COMM_ROUNDS=1000 LOCAL_EPOCHS=5 DADPFED_CYCLE=40 DADPFED_RETENTION=0.75 \
bash scripts/run_dadpfed_exp.sh
```

#### IID-like setting: $\alpha_d = 100$
```bash
DATASET=UCF101-MA METHOD=DADPFed NON_IID=1 SPLIT_COEF=100 TOTAL_CLIENT=100 ACTIVE_RATIO=0.15 \
COMM_ROUNDS=1000 LOCAL_EPOCHS=5 DADPFED_CYCLE=40 DADPFED_RETENTION=0.75 \
bash scripts/run_dadpfed_exp.sh
```

### Step 5. Full participation experiments
Set:
```
ACTIVE_RATIO=1.0
TOTAL_CLIENT=100
```
and then run the same experiment launcher.

## Output Files
Training outputs are saved under:
```
out/T=<rounds>/<dataset>-<split_rule><split_coef>-<clients>/active-<ratio>/
```

Important files include:
- `Performance/trn-<METHOD>.npy`
- `Performance/tst-<METHOD>.npy`
- `Divergence/divergence-<METHOD>.npy`

Example verified outputs:
```
out/T=1/UCF101-MA-Dirichlet0.3-10/active-0.2/Performance/tst-DADPFed.npy
out/T=1/UCF101-MA-Dirichlet0.3-10/active-0.2/Performance/tst-DADPFedSAM.npy
```

## Notes and Current Scope
- In the current framework, UCF101-MA is processed in a way that is compatible with the existing federated training pipeline.
- Hyperparameters can be edited in `scripts/run_dadpfed_exp.sh` or overridden through environment variables.
- Reproducibility is supported through code, scripts, and pipeline artifacts, but exact numerical replication may still vary across hardware, software versions, random seeds, and data preparation details.
- Raw benchmark videos are not redistributed in this repository.
